# Epub Анализатор

Локальный инструмент для анализа epub-книг. Читает книгу, передаёт её в локальную языковую модель через Ollama и создаёт текстовый файл с тремя разделами: краткое содержание, ключевые идеи и вердикт «стоит ли читать».

Всё работает без интернета и без API-ключей — модель запускается на вашем компьютере.

---

## Что получается на выходе

Для каждой книги создаётся `.txt`-файл с тремя разделами:

**Краткое содержание** — связный текст, 1900–4000 символов в зависимости от уровня детализации. По 2–5 предложений на каждую главу.

**Основные идеи** — нумерованный список из 7–10 ключевых мыслей книги, каждая по 1–2 предложения.

**Стоит ли читать** — два абзаца: вердикт (ДА / НЕТ) и объяснение на 2–3 предложения.

## Требования

- Python 3.10 или новее
- [Ollama](https://ollama.com) — локальный сервер для запуска моделей
- Модель `qwen3:14b` (~9 ГБ, скачивается один раз)

### Установка зависимостей

```bash
pip install ebooklib beautifulsoup4 ollama fastapi uvicorn python-multipart
```

### Установка модели

```bash
# Установить и запустить Ollama: https://ollama.com
ollama pull qwen3:14b
```

## Веб-интерфейс

Самый удобный способ работы с анализатором.

### Запуск

```bash
cd web
uvicorn app:app --host 0.0.0.0 --port 8000
```

Откройте браузер: **http://localhost:8000**

### Как пользоваться

**1. Загрузка книги**

Перетащите `.epub`-файл в зону загрузки или кликните по ней и выберите файл через диалог. После выбора отобразится имя файла и его размер.

**2. Настройки**

- **Язык результата** — `RU` для русского текста, `EN` для английского. Язык влияет только на результат анализа, не на язык самой книги.
- **Детализация** — `Краткая`: ~2 предложения на главу, 7 идей; `Полная`: ~5 предложений на главу, 10 идей.

**3. Анализ**

Нажмите кнопку **«Анализировать →»**. В левой панели появится прогресс-бар с текущим шагом и процентом выполнения. Анализ занимает несколько минут — модель последовательно генерирует три раздела.

Шаги обработки:
- 5% — проверка модели Ollama
- 15% — чтение epub и извлечение глав
- 30% — генерация краткого содержания
- 60% — выделение основных идей
- 80% — вынесение вердикта
- 100% — сохранение файла

**4. Результат**

По завершении появятся две кнопки: **«Смотреть результат»** открывает анализ прямо в браузере, **«Новая книга»** сбрасывает форму для следующей загрузки.

Файл с результатом сохраняется в папку `/Users/seregafrolov/Documents/Claude/v0005/` с тем же именем, что и исходный epub, но с расширением `.txt`.

**5. Библиотека**

В правой части экрана отображаются все обработанные книги. Кликните на карточку, чтобы открыть полный текст анализа. Нажмите **«↻ Обновить»**, чтобы обновить список.

## Командная строка (CLI)

Скрипт также работает напрямую из терминала.

```bash
cd scripts
python epub_summary.py <путь_к_файлу.epub> [параметры]
```

### Параметры

| Параметр | Значения | По умолчанию | Описание |
|---|---|---|---|
| `--lang` | `ru`, `en` | `ru` | Язык результата |
| `--output` | путь к папке | папка рядом с epub | Куда сохранить `.txt` |
| `--detail` | `short`, `full` | `short` | Уровень детализации |

### Примеры

```bash
# Простой запуск — результат рядом с книгой
python epub_summary.py ~/Books/book.epub

# На английском, сохранить в отдельную папку
python epub_summary.py ~/Books/book.epub --lang en --output ~/Summaries

# Полный анализ на русском
python epub_summary.py ~/Books/book.epub --detail full

# Всё вместе
python epub_summary.py ~/Books/book.epub --lang en --output ~/Summaries --detail full
```

## Устранение проблем

**«Не удалось подключиться к Ollama»**
Убедитесь, что приложение Ollama запущено. Если нет — запустите его вручную:
```bash
ollama serve
```

**«Модель qwen3:14b не найдена»**
Скачайте модель (потребуется ~9 ГБ и несколько минут):
```bash
ollama pull qwen3:14b
```

**«Не удалось извлечь ни одной главы»**
Файл может быть защищён DRM. Попробуйте другой источник того же текста.

**Результат на другом языке**
Модель иногда отвечает не на том языке — запустите повторно. Для стабильного английского используйте `--lang en`.

**Медленная генерация**
Скорость зависит от мощности GPU/CPU. На M1/M2 Mac анализ занимает 3–7 минут на книгу. Убедитесь, что Ollama использует GPU (Metal), а не CPU.
